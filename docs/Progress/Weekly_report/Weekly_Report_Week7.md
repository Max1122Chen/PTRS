#### 1.2.3 日记附件上传与回传浏览闭环

- 后端新增 `POST /api/diary/upload`，支持 JPG/PNG/MP4、大小校验、按类型落盘并回传 `url/mediaType`。
- 新增 `/media/**` 静态资源映射与安全放行，前端可直接访问上传内容。
- 前端日记编辑页改为单拖拽区上传；详情页视频由链接改为内嵌播放器。
- 前后端形成“上传 -> 回传 URL -> 提交日记 -> 详情浏览”的完整链路。

#### 1.2.4 日记提交 500 故障修复（数据库离线场景）

- 根因定位：事务创建阶段即触发数据库连接异常（`CannotCreateTransactionException` / `DataSourceDisableException`）。
- 修复策略：写接口采用“DB 优先 + dev 容错回退内存写入”，数据库不可用时不再直接 500。
- 覆盖 create/update/delete/rate 写路径，保障开发环境离线可演示能力。

#### 1.2.5 日记搜索交互升级（统一输入 + 候选直达）

- 统一输入框支持“标题/正文/目的地”检索。
- 即时候选仅展示“日记实体”，并支持点击候选直达详情。
- 单字搜索可命中标题/正文：当关键词长度 `<2` 时回退内存 contains 匹配，避免 NGram 最小粒度限制导致漏检。
- 保持课程约束：检索计算仍在内存索引与应用层完成。

#### 1.2.6 今日新增：FR-009-4 正文压缩存储落地

- 新增 `DiaryContentCodec`，采用 `RAW:` / `GZB64:` 双协议：
  - 写库按阈值压缩（GZIP + Base64）
  - 读取自动解码并兼容历史无前缀明文
- `DiaryServiceImpl` 写库路径接入编码，`InMemoryDataLoader` 预加载接入解码。
- 对外 API 与内存检索保持明文语义不变，避免影响现有搜索与展示。
- 新增配置项：
  - `app.diary.content-compression.enabled`
  - `app.diary.content-compression.threshold-bytes`
